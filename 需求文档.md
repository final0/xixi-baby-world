## 一、项目概述

| 字段 | 内容 |
|------|------|
| 项目名称 | 汐汐的小窝（xixi-baby-world） |
| 项目定位 | 专属宝宝成长记录平台，集照片、视频、日记、AI 助手于一体 |
| 目标用户 | 汐汐的家人（爸爸、妈妈、爷爷、奶奶等） |
| 访问方式 | Web 端（PC + 移动端自适应） |
| 项目仓库 | https://github.com/final0/xixi-baby-world |

### 核心功能一览

- 家庭成员注册、登录（角色制）
- 首页成长展示（生日、体重身高、精选照片、专属小短句）
- 照片墙、视频集、成长日记分区展示
- 汐汐小助手 AI 聊天（豆包/Deepseek API）
- 管理员后台（SMTP 配置、内容管理）

### 4.1 用户表 `t_user`

```sql
CREATE TABLE `t_user` (
  `id`           BIGINT       NOT NULL AUTO_INCREMENT PRIMARY KEY COMMENT '用户ID',
  `username`     VARCHAR(50)  NOT NULL UNIQUE COMMENT '登录用户名',
  `password`     VARCHAR(255) NOT NULL COMMENT 'BCrypt 加密密码',
```

## 五、模块功能详细说明

### 5.1 注册模块

**流程：**

```
用户填写 [用户名 + 邮箱 + 密码 + 邮箱验证码]
    ↓
前端调用 POST /api/auth/send-email-code（发送验证码）
    ↓
后端生成6位数字验证码，存入 Redis（key: email:code:{邮箱}, TTL: 5分钟）
    ↓
JavaMail 通过 QQ SMTP 发送至用户邮箱
    ↓
用户填写验证码，提交注册
    ↓
后端校验验证码（Redis比对），注册成功
    ↓
写入 t_user 表，first_login=1
```

**字段校验规则：**

| 字段 | 规则 |
|------|------|
| username | 4~20位，仅字母数字下划线，不能重复 |
| email | 合法邮箱格式，不能重复 |
| password | 8~20位，须含字母和数字 |
| email_code | 6位数字，5分钟内有效，验证后立即删除 |

---

### 5.2 登录模块

**流程：**

```
用户填写 [用户名 + 密码 + 图形验证码]
    ↓
前端请求 GET /api/auth/captcha 获取图形验证码图片（base64）
    ↓
后端生成算术题（如 3 + 5 = ?），答案存入 Redis（key: captcha:{captchaId}, TTL: 3分钟）
    ↓
用户提交 POST /api/auth/login
    ↓
后端验证图形验证码 → BCrypt验证密码 → 生成JWT Token
    ↓
Token 存入 Redis（key: token:{userId}, TTL: 7天），返回给前端
    ↓
前端存入 localStorage，后续请求 Header 携带 Authorization: Bearer {token}
```

**首次登录额外流程：**

```
登录成功后，检查 first_login=1
    ↓
前端跳转至角色选择页
    ↓
用户选择角色 → POST /api/user/select-role
    ↓
更新 t_user.role / role_motto / first_login=0
    ↓
跳转至首页
```

### 5.3 首页模块

首页数据由单一接口 `GET /api/home/info` 聚合返回，包含：

```json
{
  "baby": {
    "name": "汐汐",
    "birthday": "2024-01-01",
    "age": "2岁5个月",        // 后端实时计算
    "birthWeight": 3.25,
    "birthHeight": 50.0,
    "avatarUrl": "..."
  },
  "featuredPhotos": [...],     // 精选照片列表（最多9张）
  "mottos": [...]              // 今日随机短句5条
}
```

**年龄计算逻辑（后端实现）：**
- 不足1个月：显示 "X天"
- 1~12个月：显示 "X个月"
- 满1岁：显示 "X岁X个月"

---

### 5.4 照片墙模块

- **上传**：支持 jpg/png/webp/gif，单文件最大 20MB
- **存储**：上传至 MinIO，路径规则：`photos/{yyyy}/{MM}/{uuid}.{ext}`
- **缩略图**：后端上传时同步生成 400×400 缩略图（使用 Thumbnailator 库）
- **展示**：瀑布流布局（前端用 CSS columns 或 Masonry 实现）
- **分页**：每页20张，按 created_at 降序
- **管理员权限**：可删除任意照片、设置为首页精选

---

### 5.5 视频集模块

- **上传**：支持 mp4/mov/avi，单文件最大 500MB
- **存储**：上传至 MinIO，路径：`videos/{yyyy}/{MM}/{uuid}.{ext}`
- **封面**：用户可选择上传封面图，也可不传（展示默认封面）
- **展示**：卡片形式，含标题、封面、时长、上传日期
- **播放**：前端使用 `<video>` 标签，src 指向 MinIO 预签名 URL（有效期1小时）
- **分页**：每页12个，按 created_at 降序

---

### 5.6 成长日记模块

- **编辑器**：前端使用 [Quill.js](https://quilljs.com/) 富文本编辑器
- **功能**：创建/编辑/删除日记，支持心情标签选择
- **展示**：按日记日期倒序列表展示；可按月份筛选
- **权限**：每个用户只能编辑/删除自己的日记；管理员可删除所有人的

---

### 5.7 专属小短句模块

- 首页滚动展示：每次随机取5条，前端使用 CSS `animation` 滚动
- 管理员可在后台新增/删除/启用/禁用短句
- 建议初始维护不少于10条

---

### 5.8 AI 助手模块（汐汐小助手）

**入口：** 页面右下角固定悬浮圆形图标（z-index: 9999），点击展开对话抽屉。

**首次打开欢迎语：** 「我是汐汐小助手，你要干啥？😄」

**对话流程：**

```
用户输入消息
    ↓
前端 POST /api/ai/chat，携带 session_id（首次由前端生成UUID存 localStorage）
    ↓
后端从 Redis 读取该 session 近20条历史消息（key: ai:session:{sessionId}）
    ↓
组装 messages 数组（含 System Prompt + 历史 + 当前消息）
    ↓
调用 Deepseek API（POST https://api.deepseek.com/v1/chat/completions）
    ↓
返回 AI 回复，同时将 [用户消息, AI回复] 追加写入 Redis（TTL: 24小时）
    ↓
同步异步写入 t_ai_chat 表做持久化（不阻塞响应）
    ↓
前端展示 AI 回复（支持 Markdown 渲染，使用 marked.js）
```

**System Prompt（固定）：**

```

你的性格俏皮可爱、有礼貌、善解人意，非常关心家人的感受。
```

## 八、安全规范

### 8.1 接口安全

- 所有需鉴权接口必须校验 JWT Token，解析失败返回 401
- Token 刷新策略：每次请求若 Token 剩余有效期 < 1 天，自动续期并在响应 Header 返回新 Token
- 管理员接口额外校验 `is_admin=1`，否则返回 403
- 文件上传接口限制文件类型（白名单）和大小（后端校验）

### 8.2 密码安全

- 前端传输明文（HTTPS 保证传输安全），后端使用 `BCryptPasswordEncoder` 加密存储
- 禁止日志中打印密码字段

### 8.3 验证码

- 图形验证码：服务端生成，Redis 存储（TTL 3分钟），验证后立即删除，防重放
- 邮箱验证码：6位随机数字，Redis 存储（TTL 5分钟），同一邮箱60秒内限发1次
- 登录失败超过5次，锁定账号15分钟（Redis 计数）

### 8.4 MinIO 配置

- Bucket 不开放公共读写，文件访问统一走后端 API（视频播放用预签名 URL，有效期1小时）
- 照片/头像可通过 Nginx 代理对外暴露（设置合理缓存头）

```nginx
server {
    listen 80;
```
